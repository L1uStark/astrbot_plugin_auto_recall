# AstrBot 自动撤回插件

自动撤回机器人自身发出的消息。支持撤回 AstrBot 自己的回复，以及通过同一 NapCat 连接的其他客户端（如 MaiBot）发出的消息。

## ✨ 功能

- 自动撤回机器人自身消息（AstrBot + MaiBot 等）
- 延迟撤回：可设置消息发送后等待一段时间再撤回
- 关键词触发：仅撤回包含特定关键词的消息
- 最长文本触发：仅撤回纯文本长度超过设定值的消息
- 群白名单：可指定仅在部分群内生效
- 所有配置项均可在 AstrBot WebUI 中实时修改，无需重启

## 📦 安装

1. 将本插件文件夹放入 `AstrBot/data/plugins/` 下
2. 重启 AstrBot 或在 WebUI 插件页面启用 `astrbot_plugin_auto_recall`
3. 点击插件 `⚙` →  即可调整参数

## ⚙️ 配置说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `recall_time` | 浮点数 | 0.5 | 消息发送后延迟多久撤回（秒） |
| `group_whitelist` | 列表 | `[]` | 群白名单，仅在列表中的群生效；留空对所有群和私聊生效 |
| `recall_words` | 列表 | `[]` | 触发撤回的关键词列表，消息包含任一关键词即撤回；留空不限制 |
| `max_plain_len` | 整数 | 0 | 消息纯文本的最大长度，超过此长度即撤回；0 表示不限制 |

> 以上条件为“或”关系，满足其一即会撤回。

## 🧠 工作原理

- **AstrBot 自身消息**：通过 `on_decorating_result` 钩子，在回复发送前拦截，改为手动发送并延迟删除，同时阻止原始消息发出。
- **MaiBot 等外部客户端消息**：通过 `event_message_type(EventMessageType.ALL)` 监听所有到达 AstrBot 的事件，判断发送者为机器人自身时，执行同样的撤回规则。

两种方式共享相同的配置，无需额外设置。

## 📁 依赖

无外部依赖，使用 AstrBot 内置 API 和 OneBot v11 标准接口。
